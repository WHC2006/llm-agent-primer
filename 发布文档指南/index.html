    把仓库发布成「文档站」的步骤（GitHub Pages）

    你现在的仓库里只有 Markdown，在 GitHub 上点开是源码。想变成像文档站一样（左侧目录、右侧正文、可搜索），用 GitHub Pages 就行：GitHub 免费帮你生成一个网站，网址是 https://你的用户名.github.io/仓库名/。

    思路（两句话）

    1. 用 MkDocs 把当前这些 .md 转成一个带导航的静态网站（生成一堆 HTML）。
    2. 用 GitHub Actions 在你每次 git push 时自动构建并推到 gh-pages 分支；再在仓库设置里打开 GitHub Pages，选择从 gh-pages 分支发布，网站就出来了。

    你本地不用装 MkDocs，只要会 git push 即可；构建在 GitHub 服务器上跑。

    第一步：仓库里已经有什么

    本仓库里已经加好了：

    • docs/：所有文档的 Markdown 源文件放在此目录，MkDocs 从这里构建站点。
    • mkdocs.yml：告诉 MkDocs 使用 docs_dir: docs、用哪些 md 做页面、站点标题、Material 主题。
    • .github/workflows/docs.yml：每次推送到 main（或 master）时，在 GitHub 上自动执行 mkdocs build 并把结果推到 gh-pages 分支。

    所以你不需要再新建这些文件，只要按下面步骤打开 Pages 并推一次代码即可。

    第二步：在 GitHub 上打开 Pages

    1. 打开你的仓库页面，点 Settings（设置）。
    2. 左侧找到 Pages（在 “Code and automation” 下面）。
    3. 在 Build and deployment 里：
    4. Source 选 Deploy from a branch。
    5. Branch 选 gh-pages，文件夹选 /(root)。
    6. 点 Save。

    第一次可能还没有 gh-pages 分支——没关系，等你执行完第三步并 push 一次后，Actions 会自动建这个分支，然后过一两分钟再回这里刷新，分支就会出现；若已经选好 gh-pages，网站就会开始部署。

    第三步：触发一次构建（发布网站）

    1. 本地确认 mkdocs.yml 和 .github/workflows/docs.yml 已经在仓库里（你拉取最新代码或自己加过）。
    2. 正常提交并推送，例如： bash git add . git commit -m "enable docs build" git push origin main （如果默认分支叫 master，就改成 git push origin master。）
    3. 打开仓库页的 Actions 标签，应该能看到一个 workflow 在跑（名字一般是 “Deploy docs” 或你在 yml 里写的名字）。等它变绿（约 1～2 分钟）。
    4. 再回到 Settings → Pages，确认 Branch 是 gh-pages，然后点一下顶部的 Visit site，或直接打开：
      https://你的用户名.github.io/仓库名/

    网站就是你的文档站：左侧是导航（来自 mkdocs.yml 里的 nav），右侧是当前笔记内容。

    以后怎么更新文档

    • 只改 .md：改完 git add、commit、push 到 main（或 master），Actions 会自动重新构建并更新 gh-pages，几分钟后网站就更新了。
    • 要改导航：改 mkdocs.yml 里的 nav，增加或调整页面/顺序，再 push 即可。

    常见问题

    Q：点 Visit site 是 404？
    等 2～3 分钟再试；确认 Actions 里最后一次 workflow 是成功的，且 Settings → Pages 里选的是 gh-pages 分支。

    Q：网站有内容但样式错乱 / 点链接 404？
    多半是 base 地址 不对。MkDocs 在 mkdocs.yml 里有一项 site_url 或 Material 主题的 repo_url，要和你实际访问的地址一致，例如 https://你的用户名.github.io/仓库名/。仓库名要和 GitHub 上一致、结尾可有可无斜杠。

    Q：不想用 MkDocs，想用别的（如 VuePress / Docusaurus）？
    可以。思路一样：在 Actions 里改成用 VuePress/Docusaurus 构建，把生成出来的静态文件推到 gh-pages，Pages 仍然从 gh-pages 发布。本指南只写了 MkDocs 这一种，因为配置最少、和纯 Markdown 最搭。

    Q：能用自己的域名吗？
    能。在仓库 Settings → Pages 里可以填 Custom domain，然后在你域名那边按 GitHub 的说明加一条 CNAME 解析即可。

    小结

    步骤 做什么
    1 仓库里已有 mkdocs.yml 和 .github/workflows/docs.yml
    2 Settings → Pages → Source 选 “Deploy from a branch”，Branch 选 gh-pages
    3 git push 到默认分支，等 Actions 跑完，打开 https://用户名.github.io/仓库名/

    之后只要改 md 或改 mkdocs.yml 再 push，文档站就会自动更新。
