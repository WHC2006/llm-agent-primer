## Agent 的「感知→推理→行动→反馈」具体指什么？

**问：** 感知、推理、行动、反馈在 Agent 里具体指什么？每一步在代码里对应啥？

- **感知**：Agent 能「看到」的输入。通常就是当前**上下文**：用户消息、上一轮对话、以及（若有）工具返回的结果、RAG 检索到的片段等。代码里对应你拼好之后送给 LLM 的那段 prompt / messages。
- **推理**：模型根据当前上下文**决定下一步**——是直接回答用户，还是先调用某个工具、查某条数据。代码里就是一次 LLM 调用，输出里可能包含「要调用的工具名 + 参数」。
- **行动**：系统**真的去执行**推理结果里指定的动作，例如调用搜索 API、执行计算、查数据库。代码里是你根据模型返回的 tool_calls 去执行对应函数，拿到执行结果。
- **反馈**：把**行动的结果**（如搜索到的内容、计算结果）再塞回上下文，作为下一轮「感知」的一部分，让模型基于结果继续推理或给出最终回答。代码里就是把 tool 的返回值 append 到 messages，再调一次 LLM。

所以闭环就是：拼上下文（感知）→ 调 LLM 得决策（推理）→ 若有工具则执行（行动）→ 结果写回上下文（反馈）→ 再拼上下文……直到模型决定「不再调工具、直接回复用户」为止。

## ReAct 的 thought-action-observation 到底怎么转？何时用？

**问：** ReAct 的 thought-action-observation 到底怎么转？什么时候该用 ReAct？

- **Thought（思考）**：模型用自然语言简短说明「我接下来要做什么、为什么」。例如："我需要先查一下当前天气再回答。"
- **Action（行动）**：模型输出要调用的**工具名 + 参数**（如 `search(query="北京今天天气")`）。系统执行该工具。
- **Observation（观察）**：系统把**工具返回的结果**原样塞回上下文，例如 "北京今天晴，15～22℃"。模型把这段当成「观察到的现实」。
- **循环**：模型看到 Observation 后，再写下一段 Thought（如 "已有天气，可以回答穿衣建议了"），再决定是继续 Action 还是直接给出最终答案。

**何时用 ReAct？** 当任务需要**多步、且下一步依赖上一步结果**时（如先搜索再总结、先查再算再答），ReAct 的「边想边做、做完再想」很合适。若任务一步就能搞定（如只问一个简单事实），单次调用或简单工具调用即可，不必上完整 ReAct 循环。

## Plan-and-Execute 和 ReAct 有啥区别？何时用规划先行？

**问：** Plan-and-Execute 和 ReAct 有啥区别？什么时候用规划先行的方式？

- **ReAct**：不先列总计划，**边走边想**。每一步根据当前上下文（含上一步的 observation）决定下一个 thought 和 action，灵活但可能步骤冗余或遗漏。
- **Plan-and-Execute**：**先规划、再执行**。第一步让模型根据用户目标输出一个「步骤列表」（如 1. 搜索 A；2. 搜索 B；3. 对比并总结），然后再**按顺序**执行每一步（每步可能再调 LLM 或工具），不再中途改计划。

**何时用 Plan-and-Execute？** 当任务**步骤多、且结构清晰**时（如写报告要：查资料→列提纲→写各段→润色），先规划能减少「想到哪做到哪」的混乱；且便于人类审阅或调整计划。当任务**开放、路径不确定**时（如用户随口问「帮我搞定出差」），ReAct 的即兴决策往往更合适。

## Tool Use / Function Calling 在 Agent 里到底是干啥的？

**问：** Tool Use / Function Calling 在 Agent 里到底是干啥的？没有工具算不算 Agent？

- **干啥的**：让模型在「只能生成文本」之外，能**声明**「我要调用某个函数，参数是某某」。系统根据这份声明真的去执行函数（搜索、计算、查库、调 API），把结果当作文本塞回上下文，模型再基于结果继续推理或回复。没有 Function Calling 时，模型只能「假装」调用（生成一段像搜索结果的文字），无法拿到真实数据。
- **没有工具算不算 Agent？** 狭义上，常把「能主动使用工具/环境反馈」当作 Agent 的要素之一；若只有多轮对话、没有任何工具或外部反馈，多数人会叫「多轮对话系统」而非 Agent。广义上，若把「根据反馈调整下一步」也算 Agent，则纯对话也可算很弱的 Agent；但通常说 Agent 会默认带工具能力。

详见 [[工具与Function-Calling]]。
