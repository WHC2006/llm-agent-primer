## 前言与使用说明

> [!info] 用法
> - 本清单是**学习路径总纲**，按 Part 1 → Part 6 顺序学习效果更佳，也可按需跳读。
> - 每项前的 `- [ ]` 可在 Obsidian 中勾选，用于跟踪进度。
> - 带 `[[笔记名]]` 的模块可单独建笔记，后续把链接补全即可形成双向链接知识库。
> - 具体知识点请通过提问与回答沉淀到对应笔记中，本文件只列出「要学什么」。

## Part 1：AI 应用基础

详见：[[AI应用概览]]

- [ ] 区分常见 AI 应用形态：聊天机器人、智能助手、工作流自动化、Copilot 等
- [ ] 了解典型技术栈：LLM API、提示工程、上下文与 Token 的基本角色
- [ ] 理解「纯 LLM 对话」与「需要工具/外部数据」的应用边界
- [ ] 知道如何从零搭一个最简单的 AI 对话应用（单轮/多轮）

---

## Part 2：大语言模型（LLM）

详见：[[LLM与提示工程]]

- [ ] 理解 LLM 的能力与边界，以及何时需要引入 Agent 而非仅用单次调用
- [ ] 掌握提示工程基础：角色设定、任务描述、输出格式、Few-shot 示例
- [ ] 理解上下文窗口、长文本与 Token 消耗，以及成本与延迟的权衡
- [ ] 会根据场景选择合适的模型（能力、价格、延迟）
- [ ] 了解常见问题：幻觉、重复、格式不稳定及应对思路

---

## Part 3：Agent 核心

### 3.1 架构与范式

详见：[[Agent架构]]

- [ ] 理解 Agent 的构成：感知 → 推理 → 行动 → 反馈的闭环
- [ ] 掌握 ReAct：thought-action-observation 循环及何时用
- [ ] 掌握 Plan-and-Execute：先规划步骤再执行的模式及适用场景
- [ ] 理解 Tool Use / Function Calling 在 Agent 中的角色

### 3.2 推理与工具

详见：[[工具与Function-Calling]]

- [ ] 了解 CoT（Chain-of-Thought）、ReAct、反思（Reflection）等推理方式
- [ ] 会用 Function Calling 定义并调用一个工具（如搜索、计算器）
- [ ] 理解工具描述（名称、参数、说明）如何影响模型选择与调用
- [ ] 知道工具设计原则：何时拆工具、粒度、安全与权限

---

## Part 4：记忆与检索

详见：[[记忆与RAG]]

- [ ] 理解短期记忆：对话历史、上下文窗口管理与截断策略
- [ ] 理解长期记忆与 RAG：向量库、嵌入、检索流程
- [ ] 掌握检索策略：Top-K、重排序、与提示的拼接方式
- [ ] 能将 RAG 与 Agent 结合：何时检索、如何把检索结果交给 LLM
- [ ] 了解记忆的持久化与多轮会话中的一致性

---

## Part 5：多 Agent 与协作

详见：[[多Agent系统]]

- [ ] 理解多 Agent 的典型场景与角色划分（如研究员、写代码、审核）
- [ ] 掌握协调方式：顺序执行、并行、辩论、路由/调度
- [ ] 了解多 Agent 间的消息格式与状态共享
- [ ] 知道何时用多 Agent 而非单 Agent 加更多工具
- [ ] 了解主流多 Agent 框架的抽象（Crew、Team、Role）

---

## Part 6：工程化与落地

详见：[[Agent工程化]]

- [ ] 了解主流框架的定位与选型：LangChain、LlamaIndex、CrewAI、AutoGen 等
- [ ] 掌握评估维度：幻觉、延迟、成本、稳定性与可观测性
- [ ] 理解日志、追踪与监控在 Agent 调试与运维中的作用
- [ ] 了解安全与合规：输入输出过滤、权限、数据隐私与审计
- [ ] 知道常见部署形态：API 服务、异步任务、与现有系统集成
